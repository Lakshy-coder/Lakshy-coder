## &nbsp;`projects`

<details>
<summary>&nbsp;<b>gesture-trainer</b> &nbsp;·&nbsp; Browser-based hand gesture recognition using TensorFlow.js</summary>

<br/>

Built to explore what's possible with ML entirely inside a browser - no Python server, no backend. Uses **MediaPipe Hands** to read 21 hand landmarks from the webcam in real time, then trains a small classifier using **TensorFlow.js** to recognise custom gestures, all running in the browser tab itself.

I built this to understand how ML models work outside of Python notebooks - in a JavaScript runtime, talking to a webcam, with no server involved. It was messy to get working and I learned a lot from that mess.

**Stack:** JavaScript · TensorFlow.js · MediaPipe Hands · HTML5 · CSS3

[![View Repo](https://img.shields.io/badge/View%20Repository-A78BFA?style=flat-square&logo=github&logoColor=white)](https://github.com/Lakshy-coder/gesture-trainer)

</details>

<details>
<summary>&nbsp;<b>weather-app</b> &nbsp;·&nbsp; Real-time weather using a public REST API</summary>

<br/>

A weather application built with React, Tailwind CSS, Node.js, Exprees.js. Calls the OpenWeatherMap API, parses the JSON response, handles errors (invalid city names, network failures), and renders the result dynamically into the DOM.

The goal was to understand `fetch`, `async/await`, and DOM manipulation from first principles. Clean, functional, and built without shortcuts.

**Stack:** React · HTML5 · CSS3 · React · Express.js · OpenWeatherMap REST API

[![View Repo](https://img.shields.io/badge/View%20Repository-A78BFA?style=flat-square&logo=github&logoColor=white)](https://github.com/Lakshy-coder/weather-app)

</details>

<details>
<summary>&nbsp;<b>DungeonRPG</b> &nbsp;·&nbsp; Terminal RPG game written in C++</summary>

<br/>

A text-based dungeon RPG built in C++ to apply OOP and DSA concepts from coursework into something that actually runs and does something. Features character creation, an inventory system using STL containers, turn-based combat, and dungeon navigation.

Not a polished game - an intentional exercise in writing structured, object-oriented C++ and understanding what breaks when memory isn't managed carefully.

**Stack:** C++ · OOP · STL (vectors, maps, queues)

[![View Repo](https://img.shields.io/badge/View%20Repository-A78BFA?style=flat-square&logo=github&logoColor=white)](https://github.com/Lakshy-coder/DungeonRPG)
